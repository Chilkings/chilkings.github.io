stm32启动模式和下载方式

发表于 2020-07-14 | 分类于 单片机 | | 阅读次数:

启动模式

从图中可以看到, stm32 一共有三种启动模式，通过BOOT0和BOOT1进行控制。

主flash（Main Flash memory）

是STM32内置的Flash，一般我们使用JTAG或者SWD模式或者串口下载程序时，就是下载到这个里面，重启后也直接从这启动程序。

系统储存器（System memory）

从系统存储器启动，这种模式启动的程序功能是由厂家设置的。一般来说，这种启动方式用的比较少。系统存储器是芯片内部一块特定的区域，STM32在出厂时，由ST在这个区域内部预置了一段BootLoader， 也就是我们常说的ISP程序， 这是一块ROM，出厂后无法修改。一般来说，我们选用这种启动模式时，是为了从串口下载程序，因为在厂家提供的BootLoader中，提供了串口下载程序的固件，可以通过这个BootLoader将程序下载到系统的Flash中。但是这个下载方式需要以下步骤：

  • 将BOOT0设置为 1 ，BOOT1设置为 0 ，然后按下复位键，这样才能从系统存储器启动BootLoader

  • 最后在BootLoader的帮助下，通过串口下载程序到Flash中

  • 程序下载完成后，又有需要将BOOT0设置为GND，手动复位，这样，STM32才可以从Flash中启动

嵌入式SRAM(Embedded Memory)

内置SRAM，既然是SRAM，自然也就没有程序存储的能力了，这个模式一般用于程序调试。假如我只修改了代码中一个小小的地方，然后就需要重新擦除整个Flash，比较的费时，可以考虑从这个模式启动代码（也就是STM32的内存中），用于快速的程序调试，等程序调试完成后，在将程序下载到SRAM中。

下载方式

串口下载

对于STM32来说，想通过串口下载必须使用串口一（对应的引脚是PA9和PA10）并且启动模式设置为系统储存器.

  • 设置启动模式为系统储存器

上电前，设置BOOT0=1，BOOT1=0。或者是在上电后，设置BOOT0=1，BOOT1=0之后，然后按一下复位按键。这两种方式都可以让设置为系统存储启动模式，从而通过串口下载程序。

  • 串口连接上PA9和PA10

  • 使用FlyMCU下载程序

STM32串口下载软件主要两个，一个是国人开发的FlyMCU，另一个是ST官方的Flash Loader Demonstrator。Flymcu简单易用，推荐使用。
